## YYYY-MM-DD - [Timing Side-Channel in Authentication]
**Vulnerability:** The `constantTimeCompare` function in `workers/shared/auth.js` leaked the exact length of the expected token via an early return if the input lengths did not match.
**Learning:** Even when hashing inputs before comparison, an early return on length check can expose the length of the secret to attackers via a timing side-channel.
**Prevention:** To perform a truly constant-time comparison, pad both inputs to the maximum length, compute hashes, compare the hashes entirely without early return, and incorporate the length check using a constant-time operation (e.g., bitwise XOR `result |= aBytes.length ^ bBytes.length;`).
